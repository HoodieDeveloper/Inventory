1. Create a new GitHub repository and upload this project.
2. In Vercel, create a new project from this repo.
3. In Vercel Storage, create a Blob store in the same project. Vercel will add BLOB_READ_WRITE_TOKEN automatically.
4. Create a Postgres database from the Vercel Marketplace (for example Neon) and copy the DATABASE_URL.
5. In Vercel Project Settings -> Environment Variables, add:
   - DJANGO_SECRET
   - DATABASE_URL
   - DEBUG=False
   - ALLOWED_HOSTS=your-project.vercel.app
   - CSRF_TRUSTED_ORIGINS=https://your-project.vercel.app
   - BLOB_READ_WRITE_TOKEN (usually added automatically when you create Blob)
6. Set the Vercel Build Command to: bash build_vercel.sh
7. Deploy the project.
8. From your local computer, run migrations against the production database:
   - set DATABASE_URL to the same production URL
   - python manage.py migrate
   - python manage.py seed_inventory
9. Open the Vercel site and log in with:
   - admin@example.com
   - Admin@123
10. Add products with images from the admin product page. Images will upload to Vercel Blob.
